贡献 CodeIgniter

无论你是不是开发者，都有许多方法来贡献CodeIgniter 项目。

CodeIgniter是一个社区驱动的项目，接受来自社区的代码和文档的提交。 这些提交是用Issues的形式提交在GitHub上的 CodeIgniter

Issues是快速点出了错误。如果你发现了错误文件然后请检查错误在CodeIgniter几个事情：第一

  • 尚未有公开issue
  • 这个issue已经被固定（检查的发展分支，或寻找封闭的issue）
  • 是不是很明显你自己定了？
  • 如果你发现一个错误，你不确定错误，你可以启动一个新线程在CodeIgniter的issue，在 Issues section!

反馈的issue是有帮助的，但更好的方法是通过“Forking”主库,提交自己的副本， 发送一个Pull Request， 这就需要你使用版本控制系统称为Git。


不是开发者？

测试人员

我们总是需要反馈工作。大部分的开发工作都是在3版， 因此，这是最伟大的需要。如果你找到你所需要的东西，绝对是一个错误，你是一个GitHub的用户， 创建一个新的“issue”。如果你不是一个GitHub的用户，你发现一个错误，你都不知道提交， 可以在CodeIgniter论坛添加一个新讨论 Issues section!

作家

每个项目都需要很好的文档！作为用户指南的一部分 Github project , 而且还有更多的教程的空间。

自媒体

我们认为CodeIgniter这个词需要好好传播、带给有价值的项目，：） 你可以在论坛上活跃，回答问题，并通过传播 开发人员或其他用户社区内的传播CodeIgniter。

版主

论坛版主可以使用，以确保帖子留在主题， 剔除不适当的用户或评论！

设计师

建议和帮助我们的网站，用户指南和论坛的设计是随时欢迎！ 我们正在研究的主题，对于这些，将分享自己的GitHub库。

你是一个开发者？

程序员

如果你想参与帮助建立CodeIgniter的下一个版本，加入我们在 Github! 详细的 贡献指南 在用户指南中，确保您的代码符合在用户指南中，同时确保您的代码符合 我们的 代码规范, 这是正确的记录，你使用 Git-Flow 分支.

代码审查者

每个项目都需要一个核心小组的开发人员，熟悉项目的标准和规范，对 审查提出的改进和修正。到目前为止，我们已经非常幸运地进行了审查，并希望能多招一些。 代码审查者还具有“commit”代码库的权限。
